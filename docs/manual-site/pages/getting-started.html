はじめに

AutoCoderのインストールと基本的な使い方

必要条件

要件 バージョン 備考
Python 3.10以上 必須
Node.js 18以上 UI使用時に必要
Claude Code CLI 最新版 必須
Git 任意 バージョン管理に推奨

インストール

リポジトリのクローン

ターミナル
git clone https://github.com/your-repo/autocoder2.git
cd autocoder2

Python依存関係のインストール

ターミナル
cd autocoder
pip install -r requirements.txt

UI依存関係のインストール（オプション）

ターミナル
cd ui
npm install

Claude Code CLIの設定

Claude Code CLIがインストールされ、認証されていることを確認してください。

ターミナル
claude --version
claude auth

基本的なワークフロー

フェーズ1: プロジェクト作成

コマンド例
/task 新しいブログアプリを作成

自動的に以下が実行されます：

  • プロジェクトタイプの自動検出
  • ディレクトリ構造の作成
  • プロジェクトレジストリへの登録
  • UIでのブラウザ表示

フェーズ2: 計画（対話型）

オーケストレーターが「計画」タイプを検出すると、対話型のQ&Aが開始されます：

  • Q1: 何を作りたいですか？
  • Q2: 誰が使いますか？
  • Q3: 主要な機能は？

並列リサーチタスク（市場、技術、競合）が実行され、フィーチャーがカンバンに表示されます。

フェーズ3: 実装

コマンド例
/task 第1章を書く

実装フローは以下の通りです：

  1. feature_get_next() で最優先フィーチャーを取得
  2. フィーチャーを実装
  3. テスト実行（UIアプリの場合はPlaywright）
  4. feature_mark_passing() でカンバンの「Done」に移動
  5. 以前のフィーチャーを回帰テスト
  6. 3秒後に次のフィーチャーへ自動継続

フェーズ4: モニタリング

進捗確認
/status

出力例: X/Y features completed (Z%)

セッション管理

操作 方法
一時停止 Ctrl+C を押す
再開 ./start.sh を再実行（中断箇所から再開）
新規プロジェクト 初回セッションで features.db が自動作成（50-200+テストケース）
継続セッション 初期化をスキップし、直接コーディングを開始
